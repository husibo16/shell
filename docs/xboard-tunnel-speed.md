# 家用部署 Xboard 面板的公网方案速度比较

在自建 Xboard 面板时，常见的三种公网访问方案在速度方面的特点如下：

1. **申请独立公网 IP（取消 CGNAT）**
   - 数据直连，流量只经过家宽运营商网络，不经过额外的代理或中转节点。
   - 延迟和带宽都取决于家宽本身，整体速度表现最佳。

2. **使用 frp / nps 等内网穿透搭配自有 VPS**
   - 家庭网络先与 VPS 建立隧道，再由访客通过 VPS 访问，链路为“访客 → VPS → 家宽”。
   - 实际速度取决于 VPS 与访客的距离、VPS 带宽以及家宽的上行能力。通常比公网直连慢一些，但若 VPS 选址合理，可接近直连速度。

3. **使用 Cloudflare Tunnel**
   - 访问流量需要经过 Cloudflare 边缘节点转发，链路为“访客 → Cloudflare → 家宽”。
   - 受 Cloudflare 节点负载、转发策略及免费计划的限速影响，延迟和带宽一般弱于自建 VPS 方案。

**结论：** 速度最快的方案是申请独立公网 IP。其次是基于自有 VPS 的 frp / nps 反向隧道；Cloudflare Tunnel 因为要经过 Cloudflare 的转发，通常在带宽和延迟上略逊一筹。
